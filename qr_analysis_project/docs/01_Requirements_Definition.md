# 01. 需求分析与问题定义 (Requirements & Problem Definition)

## 1. 项目背景
本项目旨在开发一个自动化工具，用于分析包含二维码的图像，评估二维码的质量和特征。主要关注二维码在图片中的占比、清晰度以及颜色与背景的对比度。

## 2. 核心功能需求

### 2.1 二维码面积占比分析
- **功能描述**：计算二维码有效区域（Bounding Box）在整张图片中所占的像素比例。
- **判定标准**：
  - **大于 5%**：满足要求。
  - **小于等于 5%**：不满足要求（或需警示）。
- **计算公式**：$Ratio = \frac{Area_{QR}}{Area_{Image}} \times 100\%$

### 2.2 二维码清晰度分类
- **功能描述**：分析二维码区域的图像锐度/模糊程度。
- **分类等级**（4级）：
  1. **清晰 (Clear)**：边缘锐利，解码极其容易，Laplacian方差高。
  2. **轻度模糊 (Mildly Blurred)**：边缘有轻微虚化，但通常可解码。
  3. **中度模糊 (Moderately Blurred)**：边缘明显虚化，解码可能需要增强处理或失败。
  4. **重度模糊 (Severely Blurred)**：细节丢失严重，几乎无法识别或解码。

### 2.3 二维码颜色分类
- **功能描述**：分析二维码主体颜色（前景色）与周围背景色的相似度。
- **分类等级**（2级）：
  1. **与背景颜色不相近 (High Contrast)**：颜色差异大，易于识别（如黑白、蓝白）。
  2. **与背景颜色相近 (Low Contrast)**：颜色差异小，识别困难（如深灰配黑、浅蓝配白）。

## 3. 输入输出规范
- **输入**：图片文件（JPG, PNG等）。
- **输出**：JSON格式报告，包含上述三项指标的分析结果。
