# 09. 测试策略与验证指标 (Testing Strategy)

## 1. 测试数据集构建
为了验证算法的准确性，需要构建包含各类特征的基准数据集（Ground Truth）。

- **数据集分类**：
  - **Dataset A (Clear)**: 50张完美清晰的二维码。
  - **Dataset B (Blur Levels)**:
    - B1 (轻度模糊): 30张，通过高斯模糊核 (k=3,5) 生成。
    - B2 (中度模糊): 30张，通过高斯模糊核 (k=7,9) 生成。
    - B3 (重度模糊): 30张，通过高斯模糊核 (k=11+) 或运动模糊生成。
  - **Dataset C (Color)**:
    - C1 (高对比): 黑白、蓝白等标准配色。
    - C2 (低对比): 灰-黑、黄-白等低对比度配色。
  - **Dataset D (Size)**: 不同占比的二维码图片 (1% - 50%)。

## 2. 验证指标
- **检测率 (Recall)**：在各数据集中能成功检测到二维码位置的比例。
  - 目标：清晰组 100%，重度模糊组 > 80%。
- **清晰度分类准确率 (Accuracy)**：算法判定等级与人工/预设等级的一致性。
  - 目标：> 90%。
- **颜色判定准确率**：
  - 目标：> 95%。

## 3. 自动化测试脚本
编写 Python 脚本 `test_benchmark.py`，批量读取测试集图片，运行分析逻辑，并与文件名中标记的真值（如 `img_mild_blur_01.jpg`）进行比对，生成混淆矩阵。
